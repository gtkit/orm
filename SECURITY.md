# 安全策略

## 支持范围

当前安全修复策略如下：

- `v2`：优先接收安全修复
- `v1`：维护模式，仅接收高优先级安全修复和兼容性修复

## 漏洞报告方式

如果你发现了安全问题，请不要直接在公开 Issue 中披露细节。

建议通过以下方式私下联系维护者：

1. 邮件联系项目维护者
2. 在代码托管平台使用私密安全报告功能（如果仓库已开启）

提交时请尽量包含：

- 影响版本
- 复现步骤
- 影响范围
- 临时缓解方式
- 你建议的修复方向（可选）

## 响应目标

- 2 个工作日内确认收到
- 7 个工作日内给出初步评估
- 根据严重程度安排修复和发布

## 安全基线

本仓库当前安全基线包括：

- CI 中执行 `govulncheck`
- 敏感信息默认脱敏输出
- `orm/v2` 集成测试覆盖真实 MySQL 连接路径

## 不在本仓库直接处理的范围

以下内容通常属于接入层或平台层责任，不由本仓库单独兜底：

- 数据库账户最小权限
- 网络 ACL / 防火墙
- TLS 证书分发与轮换
- 应用层审计日志归档
- Prometheus / OpenTelemetry 后端自身的安全配置
