# go-jet Infrastructure Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a MySQL-focused `go-jet` infrastructure package that manages `*sql.DB`, transactions, and execution helpers without abstracting away `go-jet` query building.

**Architecture:** Create a new sibling package `jetorm` with isolated config and lifecycle APIs. Keep the implementation minimal: open/wrap database handles, apply pool defaults, normalize execution contexts, and expose transaction helpers around `database/sql` plus `go-jet/mysql.Statement`.

**Tech Stack:** Go 1.26, `database/sql`, `github.com/go-sql-driver/mysql`, `github.com/go-jet/jet/v2/mysql`, `testify` already present indirectly.
